1. the story

After only two years building projects, we realized that API handling is inefficient and annoying. This issue inspired us to create a Python library called ergal. The function of this library was to allow the user to store API access data in a local SQLite3 database and access it/use it quickly and easily with utility methods. This approach was effective and served as a proof of concept, leading us to create the ax-s platform.

2. the utility

Why have ten dependencies when you could have one? ax-s allows you to plug in your dependencies for access through one API. You can provide authentication information, endpoints, data targets, etc. and access those registered APIs through easy.ax-s.io/api, our simple but advanced API.

3. the goal

As cloud computing becomes the norm, dependency on external services becomes more logical and necessary for development teams. Writing handlers and clients for every one of your dependencies causes an array of issues. The goal of ax-s is to increase development efficiency, codebase readability, and project scalability by making API management and usage easy.
