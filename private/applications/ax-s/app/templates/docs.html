manage

To manage your APIs, navigate to the dashboard, and view/edit each API individually. You can add APIs, endpoints, and edit all of their attributes. That said, there are a number of rules (as of now) that must be followed in order for all of your endpoints to operate correctly.

API Setup

When you click "Add API" on the dashboard page, you will be redirected to the edit page for that new API. Fill in the name, base URL, and description fields. If authentication is required across the whole API, select whatever auth type is used from the dropdown, and then fill in the necessary information.

The name and description of your API can be any normal strings, but your base URL must have a schema (i.e. http:// or https:// prefix). It must also be a valid URL in order for any endpoints on it to work.

Endpoint Setup

On the API edit or view page, you can click "Add Endpoint" to create a new endpoint on the given API. Like the API, the name and description can be any normal string, but the path must be a valid path from the base.

For example, if your base URL is https://httpbin.org, your path must start with a / like this: /json. Similarly, if your base ends with a /, your endpoints cannot.

Headers, Parameters, Body, and Targets

All of these fields must consist of valid JSON strings. The headers, params and body fields must be encapsulated by curly braces {}, and the targets field must be encapsulated by brackets [].

access

To access these APIs, use easy.ax-s.io/api. Follow these procedures to do so.

First of all, you need to use the API token shown on the view page of the API you want to access. Build the URL as such:

https://easy.ax-s.io/api/call/[your api token]

This alone won't operate, so to access your endpoints, add the following parameter to your URL.

https://easy.ax-s.io/api/call/[your api token]?endpoint=[your endpoint name]

And finally, we need authorization. To get in the front door, use your unique user token in the headers of your request, like this.

Authorization: Bearer [your user token]

As this endpoint only accepts POST requests, you can specify instance-specific information in the request body. The body must be valid JSON, and the parameters must be headers, params or body, just like in the endpoint setup in the app.

Client Libraries

If you want an even cleaner way to access your APIs, head over to our GitHub page and check out the open-source client library options. If the language you use isn't supported, email admin@ax-s.io to get a project started.
